Introduction to Web Development

HTML

This week was an introduction Web Development and the script HTML which is used to build web pages. Html has a similar syntax to XML, making it easier for me to understand, however html has defined tag names with specific semantic meanings and behaviour.

Html5 is not based on SGML; therefore does not have DTD files for standards, support for html5's new elements are variable.[1] I try to make sure that any new element from Html5 is supported by the main browsers before using it. The W3C provides a validation tool[2] has been very useful to validate the html I write, it is akin to a compiler throwing up errors for me.

Brackets Editor

Brackets is a modern text editor that makes it easy to design in the browser. It's crafted from the ground up for web designers and front-end developers.[3] This editor makes it far easier to create and organize code, with useful features like; coloured code, code hints and general code formatting.

Introduction to HTML

Elements

HTML has predefined tags as opposed to XML and this week we have learnt quite a lot of these tags, for example the anchor tag as discussed below. HTML5 with new semantic elements makes it more difficult to look at some examples before this new version as many div elements can be replaced with the new semantic tags.

The anchor tag defines a hyperlink in HTML5. It is normally used with a property of href, although not madatory and the tag will act as a placeholder for a hyperlink.[4] Links can be relative or absolute, they join one web resource to another. Relative links are used to join resources found within the exisiting website folder.

In Html 5 the picture tag was introduced allowing an image to be mapped to a media query.[7] The picture element allows aspect or size sensitive images to be replaced by a more appropriate image for the given device size.

Introduction to CSS

Linking & Inheritance

CSS, as mentioned previously, is used to control the displaying of Html elements. A Html document can include CSS directly with the style element or inline with the style property, however to completely seperate concerns the CSS should be in a seperate stylesheet.

Many Stylesheets can be used in a Html document and over the lifetime of the html document several references to the same element may be made. The Cascading notion in CSS, gives a simple clue that the order of styling makes a difference; however more then the order controls CSS inheritance. The below factors are what determine the style for any given element.[12a]

Introduction to Responsive Design

Grid Layout

The Grid view technique was used previously in responsive design and needed a mix of Html and CSS dependency to represent a theoretical column view. Columns normally would be equidistant so that html elements would have classes like 'col-9' to denote a size of 75% in a 12 column grid view. This was a concept I did not like at all and seemed like a step back from the seperation of view concerns with CSS. [13]

The Grid Layout in CSS, coupled with media queries, allows for great customisation of Html elements completely from CSS. This is a much better technique allowing for easy control of element position and sizing while retaining and hooking onto semantic classes or ids. [14]

Mobile First

I have found that many websites use a mobile first approach to design. This is so the base content and styling work on mobiles before adding embellishments for larger devices. This can lead to websites designed with a pseudo anti-desktop pattern and often don't take advantage of the available space of larger desktop or mouse functionality. In principle mobile first approach is very logical but, with all media, the developer needs to determine the target audience of the website both present and future where possible and the opinion of the business.

Flexible Design

Flexible Images

Images can be made flexible in many different ways and the techinique depends on the type of image, format and context, and where on the page the image is placed.

An image can be 'clipped' on resizing of the page meaning that instead of overflowing by default causing a horizontal scroll bar the image is clipped. The benefit of this is that the image keeps it's original size regardless of resizing however this means that part of the image is not seen on smaller devices. This technique is best for some header images and images that are repeated multiple times.

Resizing image is another techinque which normally sets the max-width property of an image to the max width of the image so that the image quality is not ruined by upscaling. When the page size is reduced the image will then scale down respectively, this is very useful but developers need to mindful of how potentially small the image could become on small devices.

Flexible images can used in conjunction with the float property so they can be displayed next to text or another media. In the CSS the width property should be set to the percentage of the parent element it should take up, and the float property needs to be set.

The video element can be scaled in a similar fashion to images which makes flexible media very consistent in HTML5.

Media Queries

Media queries is the heart of responsive design, it allows the developer to query the screen size and then give differing CSS. The one issue I had very early on using Media Queries is that any CSS in the media query block is not more specific then out of the block. I have personally pushed my media queries to the bottom of the CSS so that if the specificity is the same the media block will override the other competeting CSS.

CSS3: RGBA

RGB is a 24 bit representation of colours. Pixel colours are made up of Red Green and Blue. Each colour represents an unsigned byte, or 8 bits of information therefore in decimal each colour can range from 0 to 255 (28-1). In Hexadecimal we can represent the colours using two Hex values for each colour channel, for example 0x0000FF would be hex for blue.

The 'A' in RGBA is the alpha channel, new to CSS3, which can specify the transparency of a colour. Unlike the other channels, the alpha channel is represented with an integer or percentage.

Summary and Reflection

Tutorial Page Summary

A recipe is nice and simple and the format of the page for many websites is quite similar, it only makes sense that you wouldn't describe how to cook something before giving the needed ingredients.

The picture of the finished item is often shown first to the user, and this needed to be true for both desktop and mobile. This thought despite being specific became more abstract with the belief that on any page the user should see content that is immediately relevant and engaging. Therefore this caused a change in the whole design of the top of the pages, the menu was changed to be horizontal on the top including the "Learning Jearnal" and an Image that was set to 90vh with a title. To avoid to much scrolling I implemented a sliding flexbox element, the element is set at 400% the width while each slide was 25% of this width, the slide positions are changed by altering the left margin and with the use of transitions this gives a nice sliding effect.

The content of the recipe page is mostly lists both unordered and ordered, it made for a much clearer page with a simple purpose thats quick and easy to use. The instructions were kept minimal trying to achieve a user being able to keep two or more instuctions in the viewpoint even on a smaller mobile, such as a iPhone5/SE 320 x 560 px viewport. This would reduce excessive scrolling and make for a easier recipe to follow on any device.

The video for the tutorial was originally placed at the top of the page, however this got moved to the bottom after realising my own behaviour when looking at other websites. In some scenarios watching a video is much easier then reading a page therefore the user may be inclined to just watch the video without ever seeing much of the actual page. Pages that are used for marketing will frown on minimal viewing of a page, it can also be argued that if a page requires a lot of scrolling to find a video when the user is looking for one then the user may just leave the page before ever finding it. The choice is really between whether getting views on the video of the page is the primary objective, if this is the case then the video being at the top of the page will lead to more plays per view. I compromised on this position using the slide technique described above the user would see the image of the pancakes and title on the landing page and with minimal scrolling would be able to navigate to the video slide with ease.

The conclusion of these points is that the style of the page should be driven by the content of the page, the way that content will be consumed by the target audience. The other driver is business, as in most aspects of software, the business objectives and targets will play a role in how a page maybe structured, obviously this should be within ethical boundries avoiding 'dark-patterns'.

Contact Page Summary

This page was quite simple to style for the most part, I did find it problematic with CSS selecting 'input' elements that had all different types that I wanted to space or style differently. I however after redisigning the page found that containing the inputs in semantically relevent divs helped reduce the need for type specific CSS selectors.

I found that dealing with a label and the input as seperate entities was difficult and could cause rigid and bloated CSS so I wrapped these in a div with a class of "form-comp" short for form component which would be a label and it's input. I also would wrap these form components in other divs with a class of "form-group" simply grouping certain components together for semantic and display purposes. This wrapping and grouping of elements made it much easier to produce the format that is currently on the contacts page and replaced the long list of single label then input on seperate lines. I later changed this design after reviewing modern form and realising many rely on the text in the placeholder to describe the needed information, therefore I removed most labels and used placeholder text instead.

HTML5 did provide for some more form attributes, I used quite a few of them in my form on the contacts page such as placeholder, required, pattern. Pattern being quite a powerful one that will get many developers who know how to use regex expressions very excited. I used pattern for validation on the phone number input as the input type 'tel' does not validate. I also found that I had to use pattern for email as the input type 'email' allowed users to type in a@c but with no "." suffix which is not correct.

The last section of my contact page is a very simple example of user consent for contact. In reality this would need a 'Terms of Service' or equivelant where the details of what user information is saved and used and what rights each party has. However my example is specificly used to highlight what I think is ethical for developers, that choices by users should be automatically set to protect the users right to privacy and the user should be the driver in giving permission to the website.

Glossary

W3C
World Wide Web Consortium - an international community where members, staff and the public work to develop web standards. [5]
HTML
Hyper-Text Markup Language - core technology for building the structure of web pages. [6a]
HTML5
Html5 is either a version of html, or the start of a 'Living Standard', a standard ever evolving but maintaining backwards capabilities of previous html. For W3C the former is true, the latter is true for WHATWG. [9]
WHATWG
Web Hypertext Application Technology Working Group - a community of people interested in evolving HTML and related technologies. Founded by Apple, Mozilla and Opera jointly. [10]
XML
Extensible Markup Language - defines encoding rules of documents for entities, made up from character data and mark up. [8]
CSS
Cascading Stylesheets - core technology for building the visual and aural layout of web pages.[6b]
Importance
Importance/ !importance - is a notation in CSS which specifies a rule that should not be overridden by other rules of inheritance. [12b]
Specificity
Specificity - is the term used for how specific a CSS selector is, selectors are scored on specificity and the highest score will be the dominate property. [12c]
  • 1000 = Inline Style
  • 100 = Style by Id
  • 10 = Style by class
  • 1 = Style by element or pseudo-element
Source Order
Source Order - When specificity is the same then the last rule will be applied to the element. [12d]
