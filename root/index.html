This week was an introduction Web Development and the script HTML which is used to build web pages. Html has a similar syntax to XML, making it easier for me to understand, however html has defined tag names with specific semantic meanings and behaviour.

Introduction to Responsive Design

Grid Layout

The Grid view technique was used previously in responsive design and needed a mix of Html and CSS dependency to represent a theoretical column view. Columns normally would be equidistant so that html elements would have classes like 'col-9' to denote a size of 75% in a 12 column grid view. This was a concept I did not like at all and seemed like a step back from the seperation of view concerns with CSS. [13]

The Grid Layout in CSS, coupled with media queries, allows for great customisation of Html elements completely from CSS. This is a much better technique allowing for easy control of element position and sizing while retaining and hooking onto semantic classes or ids. [14]

Mobile First

I have found that many websites use a mobile first approach to design. This is so the base content and styling work on mobiles before adding embellishments for larger devices. This can lead to websites designed with a pseudo anti-desktop pattern and often don't take advantage of the available space of larger desktop or mouse functionality. In principle mobile first approach is very logical but, with all media, the developer needs to determine the target audience of the website both present and future where possible and the opinion of the business.

Flexible Design

Flexible Images

Images can be made flexible in many different ways and the techinique depends on the type of image, format and context, and where on the page the image is placed.

An image can be 'clipped' on resizing of the page meaning that instead of overflowing by default causing a horizontal scroll bar the image is clipped. The benefit of this is that the image keeps it's original size regardless of resizing however this means that part of the image is not seen on smaller devices. This technique is best for some header images and images that are repeated multiple times.

Resizing image is another techinque which normally sets the max-width property of an image to the max width of the image so that the image quality is not ruined by upscaling. When the page size is reduced the image will then scale down respectively, this is very useful but developers need to mindful of how potentially small the image could become on small devices.

Flexible images can used in conjunction with the float property so they can be displayed next to text or another media. In the CSS the width property should be set to the percentage of the parent element it should take up, and the float property needs to be set.
